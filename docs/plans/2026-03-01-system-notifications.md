# System Notifications Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fire a native OS desktop notification popup every time the app creates an in-app notification.

**Architecture:** Add a single IPC channel `notify:send` in the Electron main process that calls Electron's built-in `Notification` class. Expose it via the preload bridge. Hook into the one place where all in-app notifications are created â€” `addNotification` in `FinanceContext.tsx` â€” so every notification automatically gets a system popup too. Also respect the user's per-type notification preferences stored in settings.

**Tech Stack:** Electron `Notification` API (built-in, no extra packages), existing IPC/preload pattern, React Context.

### Task 4: Call notify.send from FinanceContext

**Files:**
- Modify: `src/renderer/src/context/FinanceContext.tsx`

The `addNotification` function (around line 268) currently:
1. Guards on `userId`
2. Creates `doc`
3. Updates state with `setNotifications`
4. Calls `window.electron?.db.notifications.add(...)`

**Step 1: Add the OS notification call**

After the `window.electron?.db.notifications.add(...)` line inside `addNotification`, add:

```ts
window.electron?.notify.send(n.title, n.body)
```

The full updated `addNotification` body should look like:

```ts
const addNotification = useCallback((n: Omit<Notification, 'read'>) => {
  if (!userId) return;
  const doc: Notification = { ...n, read: false };
  setNotifications(prev => {
    if (prev.some(x => x.id === doc.id)) return prev;
    return [doc, ...prev];
  });
  window.electron?.db.notifications.add(userId, doc);
  window.electron?.notify.send(n.title, n.body);
}, [userId]);
```

> **Important:** Do NOT change the `useCallback` dependency array â€” keep it as-is (just `[userId]`).

## Notes

- `Notification.isSupported()` guards against environments where it might not work (e.g., running headless in CI).
- The `silent: false` default means a sound plays on macOS/Windows. If the user wants silent notifications this can be toggled via settings later.
- The `buildNotifications` startup notifications (bill reminders, goal milestones, monthly summary) already call `addNotification` â€” they get OS popups for free with no extra work.
- The `notifs` settings flags (`billReminders`, `goalProgress`, etc.) already exist in `UserSettings`. A future enhancement could read these settings in `addNotification` to suppress OS popups per type. Not in scope for this task.
